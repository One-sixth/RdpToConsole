# 转移远程桌面连接到本地

将远程桌面（RDP）会话断开并转移到本地控制台，无需通过远程桌面断开连接。

## 用途

在远程桌面上运行 GUI 程序时，如果直接点 X 断开连接，会导致 GUI 程序黑屏、没有显示。用本程序可以安全转移远程桌面会话到本地接管，自动断开后不会导致 GUI 程序显示黑屏。

## 原理

通过 `query user` 查询当前会话 ID，再用 `tscon.exe /dest:console` 将会话转回本地控制台。

## 版本

| 版本 | 延迟 | 说明 |
|------|------|------|
| `转移远程桌面连接到本地_10秒.exe` | 10 秒 | 默认，给切换操作留时间 |
| `转移远程桌面连接到本地_30秒.exe` | 30 秒 | |
| `转移远程桌面连接到本地_1分钟.exe` | 60 秒 | |
| `转移远程桌面连接到本地_2分钟.exe` | 120 秒 | |
| `转移远程桌面连接到本地_5分钟.exe` | 300 秒 | |

所有版本均为 x64，无需额外依赖，拷到任意 Windows 10/11 机器上直接运行。

## 使用

1. 双击对应的 exe
2. 等待倒计时结束（期间可以切换窗口、断开远程等操作）
3. 会话自动转移到本地控制台

## 编译

需要 Visual Studio 2022（安装 C++ 桌面开发工作负载）。

```bat
cd 转移远程桌面连接到本地
build.bat
```

编译产物输出到 `bin/` 目录。

## 添加新版本

编辑 `build.bat`，加一行 `call :build` 即可：

```bat
call :build 600 "转移远程桌面连接到本地_10分钟.exe"
```

## 项目结构

